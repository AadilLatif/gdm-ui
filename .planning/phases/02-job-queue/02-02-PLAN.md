## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → jobs API | Untrusted job params, model_version_id, and job IDs enter here |
| jobs API → SQLite | Job lookups and result access must be ownership-checked before read/write |
| result pointer → filesystem | Result file pointers must be treated as data, not trusted paths |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | S/T | POST /api/jobs | mitigate | Validate request models strictly and require the authenticated user to own the queued job row. |
| T-02-05 | I | GET /api/jobs/{id} and /result | mitigate | Scope every lookup to the authenticated user and return 404 on ownership mismatch. |
| T-02-06 | D | jobs API | accept | Queue submissions are always persisted to SQLite; later plans add worker throttling and backoff, while this plan keeps the API thin and deterministic. |
</threat_model>
