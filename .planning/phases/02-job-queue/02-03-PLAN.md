<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| jobs DB → worker | Worker must treat DB rows as the authoritative queue state |
| worker → solver execution | CPU-bound solver calls are untrusted inputs from queued jobs |
| worker → result storage | Result paths must be persisted only after successful execution |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-07 | D | queue worker loop | mitigate | Keep the worker off the request path and execute solver work in run_in_threadpool so one job cannot block the API loop. |
| T-02-08 | T | job row mutation | mitigate | Persist every transition in SQLite and only advance state from PENDING to RUNNING to terminal states inside the worker transaction. |
| T-02-09 | I | result_path handling | mitigate | Write result_path only after successful execution and store the pointer in SQLite rather than trusting the filesystem blindly. |
</threat_model>
